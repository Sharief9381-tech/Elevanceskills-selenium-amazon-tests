1. PROJECT OVERVIEW
================================================================

This project automates end-to-end testing of the Amazon India website (amazon.in)
using Selenium WebDriver with Java and TestNG. All 6 internship tasks are implemented
as additional test cases built on top of the training project provided by ElevanceSkills.

The automation covers real-world e-commerce scenarios including product search,
filter application, cart management, user login, profile validation, and price monitoring.

4. INTERNSHIP TASKS
================================================================

TASK 1 - PRODUCT SELECTION WITH VALIDATION
File        : Task1_ProductSelectionTest.java
Time Window : 3 PM to 6 PM only

Objective:
Select a product from Amazon search results and verify the product page details.

Rules:
  Product must NOT belong to the Electronics category
  Product name must NOT start with letters A, B, C, or D
  Test executes only between 3 PM and 6 PM

Steps:
  1. Opens Amazon India and searches for borosil glass bottle
  2. Scrolls the page to trigger lazy loading of all result cards
  3. Iterates through each result card
     Reads the product title using h2 span CSS selector
     Checks if the first character is A, B, C, or D and skips if yes
     Checks if the title contains electronics keywords and skips if yes
     Selects the first valid product
  4. Clicks the product and switches to the new tab if opened
  5. Verifies on the product page:
     Title is displayed and not empty
     Title does not start with A, B, C, or D
     Price is visible
     Availability is shown

Result: PASS
  Selected product : TintBox Borosilicate Glass Water Bottle
  Price            : Rs 789
  Availability     : In stock

TASK 6 - PRICE MONITOR WITH EMAIL ALERT
File        : Task6_PriceMonitorTest.java
Time Window : No restriction

Objective:
Monitor product price and send email alert when it drops below threshold.

Rules:
  Scrapes live price from Amazon using Selenium
  Compares with threshold of Rs 1500
  Sends Gmail SMTP alert when price drops below threshold
  Saves price history to CSV file
  Uses ScheduledExecutorService for periodic scheduling

Steps:
  1. Opens product page in Chrome
  2. Tries multiple CSS selectors to find the price element
  3. Compares scraped price with threshold
  4. Sends email via Gmail SMTP on port 465 with SSL if price is below threshold
  5. Records each check to price_history.csv with timestamp

CSV Format:
  timestamp, url, price, threshold, alertSent

Result: PASS
  Price Rs 499 scraped successfully
  Alert triggered because price is below threshold of Rs 1500
  CSV file created with 8 price check records

5. DESIGN PATTERNS USED
================================================================

PAGE OBJECT MODEL
  All page interactions are encapsulated in dedicated page classes.
  AmazonHomePage              search bar and navigation
  AmazonLoginPage             sign-in flow
  AmazonSearchProductsPage    search results
  AmazonProductDetailsPage    product page
  ShoppingCartPage            cart operations

TIME GUARD PATTERN
  Each task implements a time window check at the start of the test.
  If the current time is outside the allowed window the test prints
  a skip message and returns without executing.

EXPLICIT WAITS
  All element interactions use WebDriverWait with ExpectedConditions
  to handle dynamic page loading and avoid flaky tests.

8. HOW TO RUN
================================================================

Prerequisites:
  Java 17 or higher
  Maven 3.6 or higher
  Google Chrome browser

Setup:
  git clone https://github.com/Sharief9381-tech/Elevanceskills-selenium-amazon-tests.git
  cd Elevanceskills-selenium-amazon-tests/training_project

Update config.properties:
  URL      = https://www.amazon.in/
  USERNAME = your_amazon_email@gmail.com
  PASSWORD = your_amazon_password
  BROWSER  = chrome

Run all tasks:
  mvn test -DFORCE_RUN=true

Run individual tasks:
  mvn test -Dtest=Task1_ProductSelectionTest -DFORCE_RUN=true
  mvn test -Dtest=Task2_SearchFiltersTest    -DFORCE_RUN=true
  mvn test -Dtest=Task3_EndToEndPurchaseTest -DFORCE_RUN=true
  mvn test -Dtest=Task4_LoginProfileTest     -DFORCE_RUN=true
  mvn test -Dtest=Task5_CartTotalTest        -DFORCE_RUN=true
  mvn test -Dtest=Task6_PriceMonitorTest

Note: Remove FORCE_RUN=true for real execution.
Tests will automatically enforce their time windows.
